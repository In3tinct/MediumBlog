Deep links/Custom URIs behavior

If 2 apps have registered with the same deep links, the Android system will create a pop up to choose which app opens the Custom URI.

When the same custom URI is registered, the behaviour is unpredictable as per Apple documentation. Data leaks are possible.

Permissions (Controls access to specific device resources)

Usually defined in the AndroidManifest.xml file.

Apps can request permission for call logs or SMS messages on Android and much more.

Entitlements are used for defining permissions. A somewhat similar model to Android but more restrictions. For example no text read permission, Call logs etc.
