Kernel/Source Model

Linux-based, Open source.

Hybrid (XNU). Closed source (with some open-source components)

Bootloader (Jailbreaking/Root)

The bootloader can be unlocked. You can patch a stock image with magisk. Which Installs su binary to get root on the device.

The bootloader is locked. Jailbreak/exploit is required to get root.

Some of the exploits - checkra1n/palera1n. Exploits can be Semi tethered meaning if you restart the phone you would have to jailbreak again.

App package/Binary

The app is installed as an executable package as APK. Can be retrieved from the rooted device.

The app is installed as a MachO binary. Bundled within the .app package. Can be retrieved from jailbroken devices.

Process

Each app gets a different UID on install.

All 3p app processes run under “mobile” users. Each app is placed in its own sandbox, which is essentially a restricted file system container.

File system

The file system is Sandboxed.

Android apps are installed under /data/app

Android user files are saved under /data/data.

The file system is containerized.

iOS Application stored in -  /var/containers/Bundle/Application/UUID

User Data directory -  /var/mobile/containers/Data/Application/UUID

With each app install/resinstall a UUID is given to a specific app.

App Store

Apps can be stored from other app stores. Users need to enable the “Allow untrusted source” property for sideloading the apk.

Apps are only allowed to install from the iOS app store currently.

On a jailbroken device 3p can be installed with the Cydia store.

Sensitive Storage

KeyStore provider is used to store private Keys etc. Leverages Trusted Execution Environment. TrustyOS.

Keychain to store shareable credentials, Password etc.

Keychain API, passwords etc. stored in a sqlite file on disk, encrypted by key stored in Apple's Secure Enclave.

Runs SEPOS.

App testing/Debugging

Using an Emulator.

ADB can be used to interact with emulators.

Using a Simulator.

xcrun is used to interact with simulators.

IDB is an open source tool somewhat equivalent to ADB.

Deep links/Custom URIs behavior

If 2 apps have registered the same deep links, the Android system will create a pop up to choose which app opens the Custom URI.

When the same custom URI is registered, the behaviour is unpredictable as per Apple documentation. Data leaks are possible.

Permissions (Controls access to specific device resources)

Usually defined in the AndroidManifest.xml file.

Apps can request permission for call logs or SMS messages on Android and much more.

Entitlements are used for defining permissions. A somewhat similar model to Android but more restrictions. For example no text read permission, Call logs etc.
